# ZigBee SmartHome System

一个基于 **ZigBee + STM32 + Qt** 的智能家居监测与控制系统。

本项目由 **Qt 上位机、CC2530 ZigBee 协调器、CC2530 终端节点、STM32 控制节点** 组成，实现了环境数据采集、无线通信、灯光控制、上位机数据显示与控制等功能。

> 该项目为毕业设计实物项目，已完成系统联调与演示。

---

## 1. 项目简介

本系统面向智能家居环境监测与控制场景，主要完成以下功能：

- 采集温湿度数据
- 采集烟雾浓度数据
- 采集光照强度数据
- 实现三路 PWM 灯光控制
- 通过 ZigBee 完成终端节点与协调器之间的数据传输
- 通过串口实现协调器与 Qt 上位机通信
- 在 Qt 上位机中显示实时数据、曲线和设备状态
- 支持上位机下发控制命令

项目整体采用多节点架构：

- **CC2530 协调器**：负责 ZigBee 网络建立、数据接收与串口转发
- **CC2530 传感器终端**：负责温湿度、烟雾等环境数据采集
- **STM32 控制终端**：负责光照采集和 PWM 灯光控制
- **Qt 上位机**：负责数据显示、曲线绘制和控制命令下发

## 2. 系统架构

```mermaid
graph LR
    A[Qt 上位机] <-- 串口通信 --> B[CC2530 ZigBee 协调器]

    B <-- ZigBee 无线通信 --> C[CC2530 传感器终端]
    B <-- ZigBee 无线通信 --> D[STM32 控制终端]

    C --> C1[DHT11 温湿度采集]
    C --> C2[MQ-2 烟雾检测]

    D --> D1[BH1750 光照采集]
    D --> D2[三路 PWM LED 控制]

系统数据流大致如下：

传感器节点采集数据
        ↓
ZigBee 终端节点发送数据
        ↓
CC2530 协调器接收并转发
        ↓
Qt 上位机通过串口解析数据
        ↓
界面显示数值、曲线和报警状态

控制流大致如下：

Qt 上位机下发控制命令
        ↓
CC2530 协调器通过 ZigBee 转发
        ↓
STM32 控制节点接收命令
        ↓
执行 PWM 灯光控制

4. 软件模块
4.1 Qt 上位机

Qt 上位机主要负责：

串口数据接收
自定义协议解析
温湿度、烟雾、光照数据显示
曲线绘制
按钮控制
控制命令打包并下发

对应目录：

qt_upper_computer/
4.2 CC2530 协调器

CC2530 协调器主要负责：

ZigBee 网络建立
终端节点入网管理
接收终端节点上传数据
通过串口转发数据给 Qt 上位机
接收上位机控制命令并转发给指定终端节点

对应目录：

cc2530_coordinator/
4.3 CC2530 终端节点

CC2530 终端节点主要负责：

温湿度数据采集
烟雾数据采集
传感器数据打包
ZigBee 无线发送

对应目录：

cc2530_terminal/sensor_node/
4.4 STM32 控制节点

STM32 控制节点主要负责：

光照强度采集
三路 PWM LED 控制
接收 ZigBee 控制命令
根据手动/自动模式执行灯光控制策略

对应目录：

stm32_node/

6. 通信协议说明

本项目使用自定义数据帧完成不同模块之间的数据交互。

6.1 传感器数据上报

传感器节点周期性上报温湿度和烟雾数据。

示例数据内容包括：

温度
湿度
烟雾数字量状态
烟雾模拟量数据
报警状态位
6.2 光照数据上报

STM32 控制节点可上传光照强度数据，用于 Qt 上位机显示当前环境亮度。

6.3 控制命令下发

Qt 上位机可下发控制命令，用于控制 LED 工作模式和亮度输出。

控制模式包括：

手动控制
自动控制
报警联动控制
